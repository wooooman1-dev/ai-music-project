# AI Music Project Guide v1.2

## 1. 프로젝트 목표

Bright Music은 단순한 AI 노래 생성기가 아니라 **AI 음악 비즈니스를 위한 올인원 플랫폼**을 목표로 한다.

사용자가 음악 아이디어를 기획하고, 곡을 제작하고, 결과물과 아티스트를 관리하고, 음원을 발매하고, 성과를 분석해 다음 음악 제작에 반영하는 전체 흐름을 하나의 프로그램에서 해결할 수 있도록 한다.

장기 목표 흐름은 다음과 같다.

```text
기획
→ 제작
→ 관리
→ 발매
→ 분석
→ 다음 음악 기획
```

최종 비전은 크게 유지하되, 실제 개발은 반드시 최소 기능 제품(MVP)부터 시작한다.

---

## 2. 브랜드 포지셔닝

### 2.1 핵심 포지셔닝

> **AI 음악 비즈니스를 위한 올인원 플랫폼**

Bright Music의 경쟁력은 단순히 음악을 생성하는 데 있지 않다. 음악 제작 전후의 업무까지 하나의 연결된 워크플로우로 제공하는 것이 핵심 차별점이다.

```text
아이디어와 시장 기획
→ 가사·음악 스타일·프롬프트 제작
→ 외부 AI 음악 서비스 연동
→ 음원·버전·아티스트 관리
→ 커버·영상·발매 정보 준비
→ YouTube·DSP 발매
→ 성과 분석
→ 다음 곡 추천
```

### 2.2 초기 타깃 사용자

- 유튜버
- 음악을 만들고 싶은 일반 사용자
- AI 음악 크리에이터
- 개인 뮤지션

초기에는 전문 음반사나 대기업보다, 음악 제작 경험이 많지 않아도 혼자 음악을 만들고 발표하려는 사용자를 우선한다.

### 2.3 브랜드 슬로건

> **Create. Release. Grow.**

- Create: 음악을 기획하고 제작한다.
- Release: 완성한 음악을 세상에 발표한다.
- Grow: 성과를 분석하고 음악 활동과 비즈니스를 성장시킨다.

### 2.4 브랜드 성격

Bright Music의 제품 경험은 다음 방향을 결합한다.

- Apple의 단순하고 정돈된 경험
- Notion의 배우기 쉽고 부담 없는 구조
- Cursor의 AI 중심 작업 흐름

핵심 키워드:

- Simple
- Clean
- Smart
- Creative
- Fast

AI 기능은 복잡한 설정 화면보다 사용자가 작업하는 순간 자연스럽게 도움을 주는 방식으로 제공한다.

## 3. 현재 브랜드 구조

현재 개발 단계에서는 다음 명칭을 사용한다.

| 구분 | 명칭 | 상태 | 역할 |
|---|---|---|---|
| 회사 | Bright Entertainment | 가칭 | 전체 사업과 브랜드를 관리하는 상위 조직 |
| AI 프로그램 | Bright Music | 현재 프로그램명 | AI 음악 비즈니스 올인원 플랫폼 |
| 제품 설명 | AI Music Creator | 보조 설명 | 초기 음악 제작 기능을 설명하는 문구 |
| 슬로건 | Create. Release. Grow. | 사용 | 제품의 전체 가치와 성장 흐름 표현 |
| 음반 레이블 | Bright Records | 가칭 | AI 아티스트와 음원의 발매·유통 담당 |

```text
Bright Entertainment (Company, Temporary)
│
├── Bright Music
│   ├── AI Music Business Platform
│   └── Create. Release. Grow.
│
├── Bright Records
│   └── Record Label
│
├── AI Artists
│
└── Future Services
    ├── Publishing
    ├── Media
    └── Licensing
```

### 3.1 Bright Entertainment

Bright Entertainment는 회사의 가칭이다.

역할:

- 회사 운영
- 브랜드 관리
- AI 아티스트 사업 관리
- 음원 및 콘텐츠 사업 관리
- 라이선스·저작권 사업
- 향후 신규 서비스 확장

회사명은 공개 서비스 출시 전 도메인과 상표를 검토한 뒤 최종 확정한다.

### 3.2 Bright Music

Bright Music은 사용자가 직접 사용하는 프로그램이다.

Bright Music은 장기적으로 다음 영역을 하나의 플랫폼에서 지원한다.

- 음악 시장과 아이디어 기획
- AI 곡 기획
- AI 작사
- AI 작곡 및 보컬 생성 서비스 연동
- 음악 스타일과 프롬프트 관리
- AI 아티스트 관리
- 곡 버전과 원본 파일 관리
- 싱글·EP·앨범 관리
- 커버 이미지와 영상 제작 관리
- 발매 메타데이터 관리
- YouTube 콘텐츠 제작 관리
- 음원 유통 준비와 발매 이력 관리
- 성과 분석과 다음 곡 추천

### 3.3 Bright Records

Bright Records는 Bright Entertainment 산하 음반 레이블의 가칭이다.

역할:

- AI 아티스트 관리
- 싱글·앨범 발매
- Spotify·Apple Music·YouTube Music 등 DSP 유통
- 음원 카탈로그 관리
- 저작권 및 라이선스 기록 관리

### 3.4 명칭 사용 원칙

- Bright Music은 회사명이나 레이블명이 아니라 프로그램명으로 사용한다.
- Bright Entertainment와 Bright Records는 현재 가칭이다.
- 프로그램 개발 중 회사명과 레이블명을 코드에 고정하지 않는다.
- 최종 회사명·레이블명·도메인·상표 결정은 공개 출시 전에 진행한다.
- 향후 명칭이 변경되더라도 데이터 모델과 핵심 프로그램 구조는 영향을 받지 않아야 한다.

## 4. 제품 비전과 개발 전략

### 4.1 장기 비전

Bright Music의 최종 제품 범위는 다음과 같다.

```text
Planning
→ Creation
→ Management
→ Release
→ Distribution
→ Analytics
→ Growth
```

이를 한국어 제품 흐름으로 표현하면 다음과 같다.

```text
기획
→ 제작
→ 관리
→ 발매
→ 유통
→ 분석
→ 성장
```

### 4.2 MVP 우선 원칙

장기 비전이 크더라도 처음부터 모든 기능을 동시에 개발하지 않는다.

개발 원칙:

1. 비전은 크게 유지한다.
2. 아키텍처는 확장 가능하게 설계한다.
3. 구현은 가장 작은 MVP부터 시작한다.
4. 실제 사용과 검증이 끝난 뒤 다음 기능을 추가한다.
5. 출시·유통·분석 기능은 제작 핵심 흐름이 안정된 후 확장한다.

즉, **최종 목표는 올인원 플랫폼이지만 초기 제품은 최소 기능으로 시작한다.**

### 4.3 개발 우선순위

1. Program First
2. Product First
3. MVP First
4. Quality First
5. Review First
6. Extensible Architecture
7. Brand Finalization Later
8. Domain and Trademark Later

프로그램 안에서는 다음을 설정값으로 분리한다.

- 회사명
- 제품명
- 레이블명
- 로고
- 브랜드 컬러
- 공식 도메인
- 법적 고지 명칭

## 5. 목표 워크플로우

### 5.1 장기 전체 워크플로우

```text
프로젝트 생성
→ AI 아티스트 선택 또는 생성
→ 곡 콘셉트 입력
→ 시장·청취자·장르 분석
→ 제목 후보 생성
→ 가사 생성 및 편집
→ 음악 스타일 프롬프트 생성
→ 사용자 최종 확인
→ 외부 AI 음악 서비스에서 생성
→ 생성 결과와 원본 등록
→ 버전 비교 및 최종 음원 선택
→ 품질 검토
→ 커버·영상·Shorts 제작
→ 발매 메타데이터 생성
→ YouTube·DSP 발매 준비
→ 발매·유통 이력 관리
→ 성과 분석
→ 다음 음악 기획
```

Bright Music은 외부 음악 생성 서비스 하나에 종속되지 않는 구조를 지향한다. 초기에는 Suno 중심으로 시작할 수 있지만, 장기적으로 다른 음악 생성 Provider를 추가할 수 있어야 한다.

### 5.2 초기 MVP 워크플로우

```text
프로젝트 생성
→ 곡 아이디어와 콘셉트 입력
→ AI 기획 보조
→ 제목 후보 생성
→ 가사 생성 및 편집
→ Suno용 음악 스타일 프롬프트 생성
→ 프로젝트와 버전 저장
→ 생성 음원 등록
→ 최종 음원 선택
```

초기 MVP에서는 자동 발매, DSP 직접 유통, 고급 분석을 구현하지 않는다.

## 8. AI 아티스트 및 음악 스타일 원칙

브랜드와 아티스트는 분리해서 관리한다.

```text
Bright Records
├── Artist A: 감성 팝·드림 팝
├── Artist B: EDM·퓨처 베이스
└── Artist C: 로파이·칠 음악
```

초기에는 여러 아티스트를 동시에 운영하기보다 대표 아티스트 하나 또는 대표 음악 콘셉트 하나로 시작한다.

각 아티스트는 다음 기준을 가진다.

- 주 장르
- 보조 장르
- BPM 범위
- 보컬 성별과 톤
- 감정
- 악기 구성
- 곡 길이
- 인트로 길이
- 후렴 구조
- 가사 언어
- 금지할 스타일

이 기준은 음악 프롬프트와 품질 검토에 함께 사용한다.

## 9. 외부 생성 서비스 운영 원칙

Bright Music은 초기에는 Suno를 중심으로 사용할 수 있지만 특정 서비스에 종속되지 않는다.

각 곡마다 다음 정보를 기록한다.

- 생성 날짜
- 사용한 서비스
- 사용한 계정
- 사용한 요금제
- 프롬프트
- 가사 원본
- 생성 버전
- 다운로드 원본
- 편집 내역
- 공개 날짜
- 배포 플랫폼

수익화를 목적으로 사용할 곡은 생성 당시 서비스의 상업 이용 조건을 확인한다. 서비스 약관과 플랫폼 정책은 변경될 수 있으므로 정기적으로 다시 검토한다.

음악 생성 프롬프트는 다음 요소를 구조화해 관리한다.

```text
Genre
Mood
Tempo/BPM
Vocal type
Instrumentation
Song structure
Production style
Language
Reference-free style description
Negative constraints
```

특정 실존 가수나 곡을 그대로 모방하도록 요청하지 않는다.

## 10. 품질 및 운영 원칙

업로드 또는 발매 전 확인 항목:

- 첫 10초의 매력
- 보컬 발음과 자연스러움
- 가사 오류
- 반복 구간의 과도함
- 멜로디 전개
- 후렴의 기억성
- 음질과 볼륨
- 잡음과 깨짐
- 곡의 완결성
- 아티스트 적합성
- 기존 곡과의 유사성 위험

운영 원칙:

- 품질 기준을 통과한 콘텐츠만 발행한다.
- 공개 발행 전 사람의 검토를 기본으로 한다.
- 동일하거나 유사한 콘텐츠의 반복 생성을 차단한다.
- 생성 과정과 상업 이용 근거를 기록한다.
- 외부 서비스 연동 실패가 프로젝트 데이터 손실로 이어지지 않게 한다.
- 특정 생성 서비스에 종속되지 않는 Provider 구조를 사용한다.

초기에는 YouTube 광고 하나에만 의존하지 않고, 음원 배포와 라이선스 활용이 가능한 음악 자산 구조를 만든다.

## 12. 하지 않는 것

- 올인원 비전을 한 번에 전부 구현하려는 것
- MVP 검증 전에 발매·유통·분석 기능을 과도하게 개발하는 것
- 최종 브랜드명 결정 때문에 프로그램 개발을 장기간 중단하는 것
- 회사명·레이블명·도메인을 코드에 직접 고정하는 것
- 아티스트 정체성 없이 장르를 무작위로 섞는 운영
- 의미 없는 대량 생성
- 비슷한 음악과 영상을 반복 업로드하는 것
- 권리 확인 없이 생성 음원을 상업적으로 사용하는 것
- 실존 가수의 목소리나 스타일을 그대로 모방하는 것
- 검토 없이 완전 자동 공개하는 것
- 원본 파일과 생성 기록을 보관하지 않는 운영

## 13. 성공 기준

이 프로젝트의 성공은 기능 수나 업로드 수만으로 판단하지 않는다.

초기 MVP 성공 기준:

- 사용자가 하나의 곡 프로젝트를 처음부터 끝까지 완료할 수 있다.
- 곡 기획·가사·프롬프트가 프로젝트 안에 일관되게 저장된다.
- 생성 음원과 제작 근거를 연결해 관리할 수 있다.
- 반복 제작 시간이 실질적으로 줄어든다.
- 데이터 손실 없이 다시 열어 작업을 이어갈 수 있다.

장기 성공 기준:

- Bright Music의 실제 사용 가능성과 사용자 만족도
- 음악 제작 시간과 반복 작업 감소
- AI 아티스트의 일관성
- 음악과 콘텐츠 품질
- 스트리밍과 플랫폼 다변화
- 음악 자산의 누적
- 자동화 수준 향상
- 지속 가능한 월 수익
- Bright Music 구독 사업으로의 확장 가능성

## 14. 핵심 원칙

1. Bright Music은 **AI 음악 비즈니스를 위한 올인원 플랫폼**이다.
2. 핵심 차별점은 기획 → 제작 → 관리 → 발매 → 분석을 하나로 연결하는 것이다.
3. 슬로건은 **Create. Release. Grow.**를 사용한다.
4. 최종 비전은 크게 유지하지만 개발은 최소 기능 MVP부터 시작한다.
5. 초기 타깃은 유튜버, 음악을 만들고 싶은 사람, AI 음악 크리에이터와 개인 뮤지션이다.
6. UI는 Apple의 단순함, Notion의 친근함, Cursor의 AI 중심 경험을 지향한다.
7. Bright Entertainment는 회사 가칭이고 Bright Records는 레이블 가칭이다.
8. 프로그램 개발을 최종 도메인과 브랜드 확정보다 우선한다.
9. AI는 제작 도구이며 방향과 최종 책임은 사람이 가진다.
10. 자동 생성보다 품질 기준과 검토 흐름을 먼저 만든다.
11. 생성 기록, 원본과 상업 이용 근거를 보관한다.
12. 아키텍처는 장기 확장을 지원하되 초기 구현은 복잡하게 만들지 않는다.
